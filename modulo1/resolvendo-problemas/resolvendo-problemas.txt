O primeiro passo para resolver um problema é a atenção no enunciado, temos que ler e se preciso reler a questão, temos que saber responder perguntas do tipo 'O que o código deve fazer e quais informações são essenciais para que ele rode corretamente'.
O segundo passo é retirar as informações relevantes, a maioria das vezes o enunciado irá trazer um contexto, um localização e informações fúteis para o programador, então precisamos ter atenção, focar e separar as informações relevantes.
O terceiro passo seria analisar quais são os dados de entrada e quais serão os dados de saída e se juntar com o quarto passo que seria a o esquema visual, fica tudo mais fácil. Com o esquema visual, podemos colocar no papel as informações relevantes e realmente visualizar o escopo do código, com desenhos ou escritas, podemos organizar nossos pensamentos de uma maneira mais fácil.
O quinta passo, que eu particularmente utilizo muito, seria criar um código feio, fazer o código apenas para rodar e ver se o resultado mostrado é o que esperávamos receber e com esse código mesmo ir fazendo os testes e ver se a lógica está correta. Os testes seria o sexto passo
O último e sétimo passo seria refatorar, organizar o código, agora que já está tudo rodando, podemos ver onde conseguimos melhorar, alterar nomes de variáveis, adicionar funções, para deixarmos ele mais legível e bonito também.
Resumidamente esse são os passos essenciais para a resolução de um problema e para o desenvolvimento de um código, faço prática de todas as etapas e tem me ajudado muito a organizar os meus pensamentos.
